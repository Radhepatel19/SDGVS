# SDGVS Gramin Admin (Village/Panchayat) Portal: Comprehensive Blueprint

The Gramin Admin is the primary administrator for a specific village. This portal is designed for daily management of citizen services, grievance redressal, and information broadcasting.

---

## 1. Administrative Role & Login Flow

### A. First-Time Login & Setup Flow
1. **Receive Credentials**: Admin receives an email with a Temporary Password (generated by Super Admin).
2. **Initial Login**: Enter Email + Temporary Password.
3. **Force Password Change**:
   - System detects `is_first_login == TRUE`.
   - Redirect to "Reset Password" page.
   - Admin enters New Password and confirms it.
4. **Dashboard Entry**: Upon success, `is_first_login` is set to `FALSE`, and the main dashboard loads.

---

## 2. Page-by-Page UI Structure & Site Map

### Page 1: Village Command Center (Dashboard)
- **Inside**:
  - **Quick Action Bar**: [Post Notice], [Resolve Complaint], [Verify User], [Start Poll].
  - **Live Counters**:
    - Active Complaints (Pending/In-Progress).
    - New User Registrations (Waiting Verification).
    - Current Poll Participation % (e.g., "Solar Water Tank Poll: 65% Voted").
  - **Daily Stats**: Today's Mandi price trends and weather alerts for the village.

### Page 2: User Verification Bureau
- **Inside**:
  - **Verification Queue**: A list of all villagers who have registered but are not yet verified.
  - **Profile Check**: Click a user to see their Name, Mobile Number, and Photo.
  - **Actions**:
    - [Approve]: Instantly grants user access to all app features.
    - [Reject]: Prompts for a reason (e.g., "Invalid Village Selection") and sends a notification to the user.

### Page 3: Complaint Resolution Hub
- **Inside**:
  - **Village Worklist**: Focused ONLY on complaints from this specific village.
  - **Feature Sections**:
    - **Status Management**: Dropdown to move status: `Pending` → `In Progress` → `Resolved`.
    - **Admin Remarks**: Text area to explain action taken (e.g., "Technician sent to fix the water pipe").
    - **Multimedia Viewer**: View photos and listen to audio notes from the citizen.
    - **Resolution Rating**: View the citizen's feedback after the complaint is resolved.

### Page 4: Notice & Announcement Board
- **Inside**:
  - **Notice List**: All previous announcements for the village.
  - **Notice Creator**:
    - Fields: Title, Body (Rich Text), Category (Water/Security/General/Meeting).
    - **Text-to-Speech (TTS) Toggle**: Ensure the notice is readable by the app's voice service.
    - **Publish Logic**: Send instant push notification to all village app users.

### Page 5: Community Decisions (Polls)
- **Inside**:
  - **Active Polls**: Current voting status on village projects.
  - **Poll Creator**: 
    - Question (e.g., "Which road needs repair first?").
    - Options List (Option A, B, C...).
    - Expiry Date: When the poll should automatically close.
  - **Results**: Real-time bar charts of votes.

### Page 6: Farmer Advisory & Mandi Rates
- **Inside**:
  - **Mandi Price Editor**: Specifically update prices for crops active in this village.
  - **Local Weather Hub**: View localized forecasts and post specific advisory messages (e.g., "Cover your harvested crops - rain expected tonight").
  - **Crop Calendar Customization**: Set specific sowing/harvesting milestones for the village's primary crops.

### Page 7: Village Rewards & Leaderboard
- **Inside**:
  - **Nominee List**: List of active citizens/students/farmers based on their contribution scores.
  - **Approval Tool**: Select "Village Hero of the Month" names for the Leaderboard.
  - **Certificate Generator**: Trigger the generation of digital certificates for rewards.

### Page 8: Good News Feed (Community Feed)
- **Inside**:
  - **Story Manager**: Post positive stories, photos, and updates about village development.
  - **Engagement Monitor**: View "Likes" and "Shares" from villagers to see what stories resonate.

### Page 9: Local Emergency Services
- **Inside**:
  - **Contact Directory**: Manage phone numbers for the local Police station, Ambulance driver, and Primary Health Center (PHC).
  - **Availability Toggle**: Mark a service as "Temporary Unavailable" if needed.

## 3. Core Operational Flows

### Flow A: Complaint to Resolution
1. **Notification**: Admin receives "New Complaint #123 filed" alert.
2. **Review**: Admin reads text, checks photo, and listens to the citizen's voice note.
3. **Action**: Admin changes status to `In Progress` and types: "Technician assigned".
4. **Resolution**: Once work is done, Admin marks as `Resolved` and adds a final photo of the fix.
5. **Close**: App automatically prompts the citizen to rate the resolution quality.

### Flow B: Emergency Announcement
1. **Entry**: Admin opens Notice Board.
2. **Urgency**: Sets Category to "URGENT - WATER SUPPLY".
3. **Compose**: "Main pipe broken. Water supply suspended for 4 hours."
4. **Publish**: Immediate high-priority notification hits all devices.

---

## 4. UI Design Principles
- **Simplicity-First**: Large buttons and clear status labels (Color coded: Red=Urgent, Green=Done).
- **Mobile Responsive**: Fully usable via smartphone browser so the Gramin Admin can manage the village while on the field.
- **Visual Feedback**: Every action (Saving/Updating) shows a success checkmark and toast notification.
